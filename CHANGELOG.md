- Phase 3+4 — chaos runtimes, dual-target codegen, full pipeline ([ac96f8a](https://github.com/arthur-lonfils/Peut-Etre/commit/ac96f8ab734f89cf86a86b0a8504dcde3d5807a5))
Chaos Runtimes:
  - chaos_runtime.h: self-contained C header with PeValue tagged union,
    Math Apathy (5% ±1), Conditional Laziness (5% skip), Loop Exhaustion
    (10% after 20 iters), fuzzy equality, colored stderr warnings
  - chaos_runtime.ts: TypeScript equivalent using 'any' everywhere,
    same three chaos behaviors, PeLoopState class

  Code Generation:
  - codegen.h: shared interface for both backends
  - codegen_c.c: AST→C with chaos wrappers (_pe_add, _pe_lazy_cond,
    _pe_loop_check), two-pass for forward declarations, PeValue types
  - codegen_ts.c: AST→TypeScript with 'let x: any', same chaos function
    calls, PeLoopState scoping

  Full Pipeline:
  - --emit prints generated C or TS to stdout
  - Default mode: generate temp file → gcc/ts-node → run → cleanup
  - --no-chaos disables all chaos for deterministic output
  - --seed N seeds the C RNG for reproducible chaos
  - Runtime files resolved relative to binary or cwd
- Phase 2 — recursive-descent parser with Pratt expression parsing ([bf89844](https://github.com/arthur-lonfils/Peut-Etre/commit/bf898447c89770655d8a5af7441bb74aed50fa2e))
- AST node types as tagged union (22 node types covering all language constructs)
  - Pratt precedence-climbing for expressions (9 precedence levels)
  - Full statement parsing: var decl, assignment, if/elif/else, while, break,
    continue, return, throw, print, function definitions, function calls
  - Panic-mode error recovery with synchronization at statement boundaries
  - AST dump for debugging (--ast flag)
  - NodeList dynamic arrays for variable-length children (blocks, elif chains, args)
  - Passive-aggressive French error messages
- Phase 1 — lexer, dynamic dictionary, and DevEx tooling ([94bda44](https://github.com/arthur-lonfils/Peut-Etre/commit/94bda4443179ab8a9973664a3216e41f6a40f514))
Peut-Être transpiler foundation: C-based source-to-source compiler
  for a French esoteric language where nothing is guaranteed.

  - Lexer with 49 token types, dynamic keyword loading from dictionnaire.json
  - Minimal JSON parser for the dictionary (no external deps)
  - CLI with --lex, --lint (lazy linter), --format (anarchist formatter),
    --generate-tmLanguage, --target c|ts, --seed, --no-chaos
  - 40 French keyword mappings covering control flow, booleans, types,
    operators, async, and error handling
  - Example program demonstrating variables, print, if/elif/else, while,
    and all 5 boolean probability levels
